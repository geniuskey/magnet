# PRD: 회의 일정 조율 & 예약 자동화 시스템

## 1. 개요

### 1.1 제품명
**Meeting Scheduler AI** (내부명: 회의봇)

### 1.2 목적
자연어 기반으로 동료 일정을 조회하고, 빈 시간대와 회의실을 찾아 자동으로 회의를 예약하는 LLM 기반 시스템

### 1.3 배경
- 회의 일정 조율에 소요되는 시간 절감 필요
- 여러 시스템(캘린더, 회의실 예약, 조직도)을 오가는 번거로움 해소
- 사내 인트라넷 환경에서 보안을 유지하며 운영

### 1.4 운영 환경
- **배포**: 사내 Linux 서버 (self-hosted)
- **네트워크**: 인트라넷 전용 (외부 클라우드 서비스 미사용)
- **보안**: 사내 인증 시스템 연동

## 2. 사용자 및 이해관계자

### 2.1 주요 사용자
| 사용자 유형 | 설명 | 주요 니즈 |
|------------|------|----------|
| 일반 직원 | 회의를 주선하는 모든 임직원 | 빠른 회의 일정 조율 |

| 비서/어시스턴트 | 임원 일정 관리 | 복잡한 일정 조율 자동화 |

### 2.2 이해관계자
- IT 인프라팀: 서버 운영 및 보안
- 정보보안팀: 데이터 접근 권한 관리
- 총무팀: 회의실 관리 정책

## 3. 핵심 기능 요구사항

### 3.1 자연어 회의 요청 (P0 - 필수)
**설명**: 사용자가 자연어로 회의 요청을 입력하면 시스템이 의도를 파악

**입력 예시**:
- "내일 오후에 김철수, 이영희랑 1시간 회의 잡아줘"
- "다음 주 중으로 개발팀 전체 회의 가능한 시간 찾아줘"
- "이번 주 금요일 오전에 빈 회의실 있어?"

**파싱 항목**:
- 참석자 (이름, 부서, 팀 전체 등)
- 희망 일시 (특정일, 기간, 시간대)
- 회의 시간 (duration)
- 회의 목적/제목 (선택)
- 회의실 요구사항 (인원, 장비 등)

### 3.2 직원 검색 및 일정 조회 (P0 - 필수)
**설명**: 조직 API를 통해 직원을 검색하고 캘린더 API로 일정 조회

**기능**:
- 이름/부서/직급으로 직원 검색
- 동명이인 처리 (부서 정보로 구분)
- 지정 기간 내 일정 조회
- 다중 참석자 일정 동시 조회

### 3.3 빈 시간대 탐색 (P0 - 필수)
**설명**: 모든 참석자가 가능한 공통 빈 시간대 계산

**기능**:
- 참석자 전원의 빈 시간대 교집합 계산
- 업무 시간 필터링 (기본: 09:00-18:00)
- 점심시간 제외 옵션 (12:00-13:00)
- 선호 시간대 우선 정렬 (오전/오후)

### 3.4 회의실 검색 (P0 - 필수)
**설명**: 조건에 맞는 예약 가능한 회의실 탐색

**기능**:
- 시간대별 예약 가능 회의실 조회
- 인원수 기반 필터링
- 위치(층/건물) 기반 필터링
- 장비 요구사항 필터링 (화상회의, 화이트보드 등)

### 3.5 옵션 제안 및 확인 (P0 - 필수)
**설명**: 가능한 옵션을 제시하고 사용자 선택을 받음

**기능**:
- 최대 5개 추천 옵션 제시
- 각 옵션에 시간, 회의실, 특이사항 표시
- 번호 또는 자연어로 선택 가능

### 3.6 회의 생성 및 예약 (P0 - 필수)
**설명**: 확정된 옵션으로 회의 생성 및 회의실 예약 실행

**기능**:
- 캘린더에 회의 일정 생성
- 참석자에게 초대 발송
- 회의실 예약 시스템에 예약 등록
- 예약 완료 확인 메시지 반환

### 3.7 예약 조회/수정/취소 (P1 - 중요)
**설명**: 기존 예약 관리 기능

**기능**:
- 내가 만든 회의 목록 조회
- 회의 시간/장소 변경
- 회의 취소 및 참석자 알림

### 3.8 대화 컨텍스트 유지 (P1 - 중요)
**설명**: 멀티턴 대화에서 컨텍스트 유지

**예시**:
```
사용자: 다음 주 김철수랑 회의 잡아줘
AI: 월요일 14시, 화요일 10시가 가능해요
사용자: 화요일로 해줘
AI: (컨텍스트 유지하여) 화요일 10시로 예약할게요
```

## 4. 비기능 요구사항

### 4.1 성능
| 항목 | 요구사항 |
|------|---------|
| 응답 시간 | 일반 조회 < 3초, 복잡한 조율 < 10초 |
| 동시 사용자 | 최소 50명 동시 접속 |
| 가용성 | 업무 시간 내 99% 이상 |

### 4.2 보안
- 사내 SSO/LDAP 연동 인증
- API 호출 시 사용자 권한 확인
- 타인 일정 조회 시 공개 범위 준수
- 모든 API 통신 HTTPS (사내 인증서)
- 감사 로그 기록

### 4.3 운영
- 사내 Linux 서버 Docker 컨테이너 배포
- 로그 중앙 집중화 (사내 로그 시스템 연동)
- 헬스체크 및 모니터링

## 6. 제약사항

### 6.1 기술적 제약
- 외부 클라우드 서비스 사용 불가 (Vercel, Cloudflare 등)
- LLM API는 사내 프록시 또는 self-hosted 모델 사용
- 사내 API 스펙에 의존

### 6.2 정책적 제약
- 타인 상세 일정 내용은 비공개 (바쁨/가능 여부만 표시)
- 회의실 예약 정책 준수 (최대 예약 시간 등)
