# FAQ

자주 묻는 질문과 답변입니다.

## 일반

### Meeting Scheduler AI란 무엇인가요?

Meeting Scheduler AI는 LLM(Large Language Model)을 활용하여 자연어로 회의 일정을 조율하고 회의실을 예약할 수 있는 시스템입니다. "김철수, 이영희와 내일 오후 2시에 본관 대회의실 예약해줘"와 같은 자연어 명령으로 회의를 예약할 수 있습니다.

### 어떤 LLM을 지원하나요?

현재 다음 LLM 프로바이더를 지원합니다:

- **Anthropic Claude** (권장)
- **OpenAI GPT-4**
- **Google Gemini**

환경 변수 `LLM_PROVIDER`로 선택할 수 있습니다.

### 사내 시스템과 어떻게 연동하나요?

Adapter 패턴을 사용하여 연동합니다:

1. [직원 조회 API](../api/employee-api.md) - HR/조직도 시스템
2. [일정 조회 API](../api/calendar-api.md) - 캘린더 시스템
3. [회의실 API](../api/room-api.md) - 회의실 예약 시스템

각 가이드를 참고하여 Adapter를 구현하면 됩니다.

## 설치 및 설정

### Python 버전 요구사항은?

Python 3.10 이상이 필요합니다. 3.11을 권장합니다.

### Docker로 실행할 수 있나요?

네, Docker Compose를 지원합니다:

```bash
docker-compose up -d
```

자세한 내용은 [설치 가이드](../getting-started/installation.md)를 참고하세요.

### LLM API 키 없이 테스트할 수 있나요?

네, Mock LLM 모드를 사용할 수 있습니다:

```bash
USE_MOCK_LLM=true
```

단, 이 모드에서는 실제 자연어 처리가 아닌 미리 정의된 응답만 반환됩니다.

### 외부 API 없이 테스트할 수 있나요?

네, Mock Adapter를 사용하면 됩니다:

```bash
USE_MOCK_ADAPTERS=true
```

가짜 직원, 회의실 데이터로 시스템을 테스트할 수 있습니다.

## 인증

### SSO 연동이 필수인가요?

필수는 아닙니다. 다음 옵션 중 선택할 수 있습니다:

1. **SSO (SAML 2.0)** - 대기업 환경에 권장
2. **OAuth 2.0 / OIDC** - 클라우드 IdP 사용 시
3. **인증 비활성화** - 개발/테스트 환경

### 어떤 IdP를 지원하나요?

- Microsoft ADFS
- Azure AD
- Okta
- OneLogin
- Google Workspace (OIDC만)

자세한 설정은 [SSO 연동 가이드](../guides/sso-integration.md)를 참고하세요.

### 세션 타임아웃을 변경할 수 있나요?

네, 환경 변수로 설정합니다:

```bash
SESSION_TTL=28800  # 8시간 (초 단위)
```

## 기능

### 몇 명까지 참석자를 추가할 수 있나요?

기술적 제한은 없지만, UI 사용성을 위해 다음을 권장합니다:

- 개별 선택: 20명 이내
- 팀/그룹 선택: 제한 없음

대규모 회의는 팀 또는 그룹 단위로 추가하는 것이 효율적입니다.

### 반복 회의를 설정할 수 있나요?

네, 다음 반복 패턴을 지원합니다:

- 매일
- 매주
- 격주
- 매월

반복 종료일을 반드시 설정해야 합니다.

### 예약을 대리로 할 수 있나요?

네, 참석자 유형을 구분합니다:

- **주관자**: 회의를 주최하는 사람
- **필수 참석자**: 반드시 참석해야 하는 사람
- **선택 참석자**: 선택적으로 참석하는 사람

본인이 아닌 다른 사람을 주관자로 설정할 수 있습니다.

### 회의실 장비로 필터링할 수 있나요?

네, API 수준에서 지원합니다:

```
GET /floors/{floor_id}/rooms?equipment=projector,video_conference
```

프론트엔드 UI는 향후 업데이트 예정입니다.

### 최적 시간 추천은 어떻게 작동하나요?

선택된 참석자들의 캘린더를 분석하여 모두가 가능한 공통 시간대를 찾습니다:

1. 각 참석자의 Free/Busy 정보 조회
2. 요청된 회의 시간만큼 연속으로 비어있는 시간대 탐색
3. 업무 시간(09:00-18:00) 우선순위 적용
4. 최대 5개 추천 시간 반환

## AI 채팅

### 어떤 명령어를 사용할 수 있나요?

자연어로 자유롭게 대화할 수 있습니다. 예시:

- "김철수, 이영희와 내일 오후 2시에 본관 대회의실 예약해줘"
- "개발팀 전체 회의 잡아줘. 1시간짜리"
- "다음 주 월요일 프로젝트 A팀 주간 회의 예약해줘"
- "가능한 시간대 추천해줘"

### 빠른 적용 버튼이 뭔가요?

AI 응답에서 건물, 층, 회의실, 참석자 등이 언급되면 해당 내용을 클릭 한 번으로 UI에 적용할 수 있는 버튼이 표시됩니다.

### 대화 내역이 저장되나요?

현재 세션 동안만 유지됩니다. 브라우저를 닫거나 새로고침하면 대화가 초기화됩니다. 대화 내역 저장 기능은 향후 업데이트 예정입니다.

## 문제 해결

### "API 연결 실패" 오류가 발생해요

1. 백엔드 서버가 실행 중인지 확인
2. `VITE_API_URL` 환경 변수가 올바른지 확인
3. CORS 설정 확인

### "인증 실패" 오류가 발생해요

1. SSO/OAuth 설정 확인
2. IdP에서 애플리케이션이 활성화되어 있는지 확인
3. Redirect URI가 정확한지 확인

### 회의실 예약이 안 돼요

1. 해당 시간에 이미 예약이 있는지 확인
2. 회의실이 활성화 상태인지 확인
3. 예약 권한이 있는지 확인

### LLM 응답이 느려요

1. LLM API 상태 확인
2. 네트워크 연결 상태 확인
3. `LLM_TIMEOUT` 값 조정

```bash
LLM_TIMEOUT=60  # 초 단위
```
